# Wilson 时间管理助手 ⏰

用**数据驱动**的方法管理你的申请季，让每一分钟都有价值！

## 🎯 项目目标

帮助Wilson在申请季合理分配时间，通过数据统计和可视化：
- 📊 追踪每日时间分配
- 🎯 对标IELTS分数制定学习计划
- 📈 周报分析：计划 vs 实际
- 💡 智能洞察和调整建议

## 📋 IELTS成绩分析

| 科目 | 成绩 | 状态 |
|------|------|------|
| Overall | 6.0 | 需要提升 |
| Reading | 6.5 | 较好 |
| Listening | 6.5 | 较好 |
| Writing | 6.0 | 需要加强 |
| Speaking | 5.5 | ⚠️ 最弱项 |

**建议重点**：增加口语和写作的练习时间

## 🎯 目标时间分配

```
🎯 雅思准备      → 40% ⭐ 最高优先级
✍️ 申请文书       → 20%
🏫 学校任务       → 20%
🎮 休息娱乐       → 10%
📌 其他          → 10%
```

## ✨ 功能特性

### 1️⃣ 每日时间记录
- 快速添加每天的学习时间
- 支持5个任务类别
- 添加详细备注（如：雅思阅读练习题）
- 单日任务总览

### 2️⃣ 数据可视化
- **饼图**：直观看到时间分布
- **柱状图**：对比各任务耗时
- 实时更新，即刻反映变化

### 3️⃣ 周报分析
- 📊 本周目标 vs 实际对比
- 🎯 每个任务的进度百分比
- 📈 差异分析和趋势预测
- 💡 基于IELTS成绩的个性化建议

### 4️⃣ 数据管理
- 💾 本地存储（浏览器localStorage）
- 📥 导出JSON格式数据备份
- 📤 导入历史数据恢复
- 🗑️ 清空数据重新开始

### 5️⃣ 智能洞察
- 根据IELTS弱项给出建议
- 检测学习时间是否充足
- 提醒休息平衡
- 周进度预测

## 📱 使用流程

```
1. 打开应用
   ↓
2. 在"新增记录"标签填写：
   - 日期
   - 任务类别（选择IELTS/文书/学校/休息/其他）
   - 时间（分钟）
   - 备注（可选）
   ↓
3. 点击"✅ 添加记录"
   ↓
4. 在"今日总览"看当日汇总
   ↓
5. 在"时间分配可视化"看全周分布
   ↓
6. 在"周报分析"看目标达成情况和智能建议
   ↓
7. 每周导出数据做备份
```

## 📊 周报使用建议

**每周五晚上** 的推荐操作：
1. ✅ 检查本周的目标达成度
2. 📝 在"详细数据"标签记录偏差原因
3. 📥 导出本周数据备份
4. 💡 根据"本周洞察"调整下周计划
5. 📊 对比多周数据，发现模式

### 示例分析

**情况A**：IELTS准备只有25%
```
⚠️ 问题：口语练习不足
💡 解决：下周增加口语时间，从每天10分钟增加到30分钟
🎯 目标：冲刺40%
```

**情况B**：休息时间超过20%
```
⚠️ 问题：可能精力不足或拖延
💡 分析：检查学习质量，不是时间越多越好
🎯 目标：保持10-15%，提高学习效率
```

## 🎓 学习建议

基于你的IELTS成绩，这个app会自动：

✅ **强化口语练习**（当前最弱，5.5分）
- 建议周投入：8-10小时
- 主要方向：日常对话、学术讨论、复述

✅ **巩固写作**（6.0分，有进步空间）
- 建议周投入：6-8小时
- 主要方向：大作文、小作文、词汇积累

✅ **保持阅读和听力**（已较好，6.5分）
- 建议周投入：6-8小时
- 主要方向：保持做题手感、扩大词汇

## 💾 数据保安全吗？

✅ **完全本地存储**
- 所有数据保存在你的浏览器（localStorage）
- 不上传到任何服务器
- 不同浏览器/设备之间数据独立
- 清除浏览器数据会丢失记录（建议定期导出备份！）

## 🎯 进阶用法

### 导入历史数据
如果你有之前的时间记录（Excel/其他格式），可以：
1. 转换成JSON格式（参考上面的示例）
2. 使用"导入数据"功能恢复历史记录
3. 继续追踪长期进度

### 自定义目标
编辑 `app.js` 中的 `GOALS` 对象：
```javascript
const GOALS = {
    IELTS: 35,      // 改为35%
    Writing: 25,    // 改为25%
    School: 20,
    Rest: 10,
    Other: 10
};
```

### 添加新任务类别
1. 在 `GOALS` 中添加新类别和目标百分比
2. 在 `TASK_COLORS` 中添加颜色
3. 在 `TASK_LABELS` 中添加标签
4. HTML中添加新的 `<option>` 标签
